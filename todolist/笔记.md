### 1. 标签定义
`<main>` 元素用于呈现文档或应用中**独一无二**的主体部分。

---
### 2. 项目实战结构 (以 QG 第一周作业为例)
在你的技术本（TodoList）项目中，`<main>` 标签包裹了所有核心业务逻辑：

### 3. 为什么要用 `<main>` 而不是 `<div>`？
1.  [cite_start]**无障碍访问 (Accessibility)**：辅助设备（如屏幕阅读器）可以识别该标签，让用户一键跳转到核心内容。 [cite: 36]
2.  [cite_start]**SEO 优化**：搜索引擎会给予 `<main>` 内部内容更高的权重。 [cite: 36]
3.  [cite_start]**代码可读性**：结构化标签让开发者能迅速定位到核心功能模块。 [cite: 36]

# 2.`<textarea>`

### 1. 标签定义
`<textarea>` 是一个用于输入**多行文本**的表单元素 。在“技术本”作业中，它主要用于承载笔记的具体内容，支持用户输入较长的技术心得或代码片段 。

---

### 2. `<textarea>` vs `<input>` 深度对比
根据作业要求，标题通常较短，而内容涉及详细记录，因此需要区分使用

| 特性 | `<input type="text">` | `<textarea>` |
| :--- | :--- | :--- |
| **行数限制** | 仅限单行 | **支持多行输入** |
| **回车键行为** | 通常触发表单提交 | **产生换行符** |
| **适用场景** | 笔记标题 | 笔记内容  |
| **闭合方式** | 单标签 `<input>` | **双标签 `<textarea></textarea>`** |
| **尺寸调节** | 固定高度 | 默认右下角可拖拽调节大小 |

---

### 3. 核心关键属性
为了实现作业二中的记事功能，你需要在 HTML 中配置以下属性 ：

* **`id`**: 必填项。用于 JavaScript 获取用户输入的文本内容，实现记事功能 [cite: 39]。
* **`placeholder`**: 设置未输入时的提示文字，例如“请输入笔记内容...” 
* **`rows`**: 设置默认显示的可见行数，控制初始高度。
* **`cols`**: 设置横向宽度（通常在 CSS 中用 width 替代）。

---

# 📝 HTML5 核心标签笔记：`<aside>`

### 1. 标签定义
`<aside>` 元素表示一个页面区域，其内容与页面的主要内容（通常在 `<main>` 中）间接相关。它通常被看作是“侧边栏”或“附属信息区”。

---

### 2. 核心作用
* **功能分离**：将辅助性功能（如搜索、分类、快捷操作按钮）从核心业务逻辑中抽离出来。
* **语义化结构**：明确告诉浏览器和搜索引擎，这部分内容是“附加”的，即使移除它，页面的主体功能依然完整。
* **提升无障碍体验**：屏幕阅读器会识别这是一个辅助区域，允许用户快速跳过它直达主体内容。

---

### 3. 项目实战分析 (QG技术本)
在你提供的图片中，右侧的小挂件就是典型的 `<aside>` 场景：

| 放置在 `<aside>` 中的组件 | 作用描述 |
| :--- | :--- |
| **状态切换按钮 (全部/回收站)** | 对主列表进行过滤操作，不属于笔记内容。 |
| **一键操作 (清除全部/标记完成)** | 全局性的辅助管理功能。 |
| **导入/导出 (JSON/TXT)** | 数据持久化与外部交互的进阶功能。 |

---

### 4. 常用 CSS 布局实现
为了实现图片中那种“左主右辅”的效果，通常将 `<aside>` 配合 Flexbox 布局：

```css

    gap: 20px;             /* 设置主栏与侧边栏的间距 */
}

aside.right-panel {
    width: 200px;          /* 给侧边栏一个固定宽度 */
    flex-shrink: 0;        /* 防止在窗口缩小时侧边栏被挤压变形 */
}
```

# 🎨 CSS 核心属性笔记：`gap`

### 1. 标签定义
`gap` 属性（以前叫 `grid-gap`）用于设置弹性盒子（Flex）或网格（Grid）布局中**行与行**、**列与列**之间的间隙。

---

### 2. 为什么用 `gap` 而不是 `margin`？
在 `gap` 出现之前，我们通常用 `margin` 来拉开间距，但 `margin` 有个麻烦：**最边缘的元素也会带上外边距**，导致布局不居中。

* **`gap` 的优势**：它**只作用于子元素之间**。第一个元素的前面和最后一个元素的后面不会有间距。这完美解决了“去第一个或最后一个边距”的烦恼。

---

### 3. 语法拆解
你可以设置一个值，也可以设置两个值：

| 语法示例 | 效果描述 |
| :--- | :--- |
| **`gap: 20px;`** | 水平（列）和垂直（行）间距都是 20px。 |
| **`gap: 10px 30px;`** | 上下行间距 10px，左右列间距 30px。 |

---

### 4. 项目实战分析

1. **主页面左右布局**：
   在 `<main>` 容器中使用 `gap: 20px;`，可以让左边的笔记列表和右边的功能栏保持整齐的距离。
2. **笔记列表内部**：
   给 `#note-list` 容器设置 `gap: 15px;`，每条笔记卡片之间就会自动产生均匀的呼吸感。

```css
/* 示例代码 */
.content-wrapper {
    display: flex; /* 必须先开启 flex 或 grid */
    gap: 25px;    /* 左右两栏间隔 25px */
}
```
# 🎨 CSS 核心属性：`flex` (伸缩比例)

### 1. 属性定义
`flex` 是 `flex-grow` (放大), `flex-shrink` (缩小) 和 `flex-basis` (基准大小) 的简写。但在初学阶段，你通常可以将它理解为**占份数**。

---

### 2. 它是如何工作的？(分蛋糕原理)
当父容器（如你的 `<main>`）开启了 `display: flex` 后，它会计算所有子元素的 `flex` 值总和。

以你的代码为例：
* **左侧 (`.left-panel`)**: `flex: 3;`
* **右侧 (`.right-panel`)**: `flex: 1;`
* **总份数**: `3 + 1 = 4` 份。

**结果**：
* 左侧面板会占据父容器剩余空间的 **3/4 (75%)**。
* 右侧面板会占据父容器剩余空间的 **1/4 (25%)**。

---

### 3. 常见用法对比

| 取值 | 描述 | 适用场景 |
| :--- | :--- | :--- |
| **`flex: 1;`** | 自动撑满所有剩余空间。 | 如果只有 1 个子元素设为 1，它会占满整行。 |
| **`flex: 0;`** | 不参与分配剩余空间。 | 侧边栏需要固定宽度，不随窗口拉伸。 |
| **`flex: 3;`** | 相对比例。 | 让主内容区（笔记区）明显宽于功能区。 |

---

### 4. 项目实战(左右侧边栏经常用到)
在你的左右布局中，使用 `flex: 3` 和 `flex: 1` 是非常合理的做法：

```css
main {
    display: flex; /* 必须开启 */
}

.left-panel {
    flex: 3; 
    background-color: #fff;
    border: 3px solid #000; /* 配合硬阴影风格 */
}

.right-panel {
    flex: 1;
    /* 右侧通常放按钮，不需要太宽 */
}
```
# 🎨 CSS 核心属性笔记：`border-top`

### 1. 标签定义
`border-top` 是一个复合属性，用于一次性设置元素顶部边框的 **宽度 (width)**、**样式 (style)** 和 **颜色 (color)**。

---

### 2. 语法拆解
以你的项目风格为例：`border-top: 3px solid #000;`

| 参数 | 示例值 | 作用描述 |
| :--- | :--- | :--- |
| **宽度 (width)** | `3px` | 边框的粗细。 |
| **样式 (style)** | `solid` | 边框的类型（`solid` 实线, `dashed` 虚线, `dotted` 点线）。 |
| **颜色 (color)** | `#000` | 边框的颜色。 |

> **注意**：这三个值的顺序可以随意摆放，但通常习惯按“粗细-形状-颜色”来写。

---

### 3. 为什么在项目中使用它？
在你的“技术本”中，`border-top` 有两个非常实用的场景：

* **分割线效果**：如果你想在笔记列表的每条笔记之间加一根横线，但不想给整条笔记加框，就可以只给底部的笔记项加 `border-top`。
* **卡片装饰**：给卡片顶部加一个不同颜色的粗边框（例如粉色），可以做出类似“书签”或者“文件夹分类选项卡”的视觉效果。

---

### 4. 属性拆解（如果只想改其中一项）
如果你已经设置了 `border-top`，但后面只想改颜色，可以使用子属性：
* `border-top-width`: 只改粗细。
* `border-top-style`: 只改形状。
* `border-top-color`: 只改颜色。

---

### 5. 项目实战演示

# 🛠️ JavaScript 核心方法笔记：`trim()`

### 1. 什么是 `trim()`？
`trim()` 是 JavaScript 字符串（String）对象的一个内置方法。
它的唯一任务是：**移除字符串两端的空白字符**，并返回一个新字符串，而不会修改原始字符串。

---

### 2. 它能处理哪些“空白”？
`trim()` 不仅仅能去掉空格，它还能识别并切除以下字符：
* **空格** (Space)
* **制表符** (Tab, `\t`)
* **换行符** (Newline, `\n`)
* **回车符** (CR, `\r`)



---

### 3. 为什么在 TodoList 项目中必须用它？

#### 场景 A：防止“空”笔记
如果用户只按了几个空格就点提交，没有 `trim()`，程序会认为 `"   "` 是有内容的，从而生成一张空白卡片。
* **解决**：`if (input.value.trim() === "") { ... }`

#### 场景 B：保持数据整洁
用户输入时可能不小心在末尾多打了一个空格（例如 `"买牛奶 "`）。
* **结果**：存入数据库或 LocalStorage 的数据会带有无意义的空格。
* **解决**：在存入数组前执行 `trim()`，确保数据纯净。

---

### 4. 代码实战演练

```javascript
// 模拟用户输入
let userInput = "   学习 JavaScript   \n";

// 执行清理
let cleanedInput = userInput.trim();

console.log("原始长度:", userInput.length);    // 长度包含空格和换行
console.log("清理后长度:", cleanedInput.length); // 长度仅包含文字
console.log("清理后内容:", "|" + cleanedInput + "|"); // 输出: |学习 JavaScript|

// 注意：它不会影响文字中间的空格！
let midSpace = "  Hello   World  ".trim();
console.log(midSpace); // 输出: "Hello   World"
```

# 💾 浏览器“小口袋”：数据持久化 (Data Persistence) 核心笔记

在前端开发中，让数据在页面刷新、浏览器关闭后依然存在的行为，被称为 **数据持久化**。

---




## 一、 核心术语表（专业版 vs 大白话）

| 专业术语 | 英文原文 | 解释 (Definition) | 你的代码体现 |
| :--- | :--- | :--- | :--- |
| **本地存储** | `localStorage` | 浏览器提供的一种持久化存储机制，数据除非手动删除，否则永不失效。 | `localStorage.getItem(...)` |
| **键值对** | `Key-Value Pair` | 存储数据的基本形式。`Key` 是钥匙（名字），`Value` 是锁（内容）。 | `'my_todo_data'` 就是 Key |
| **序列化** | `Serialization` | 将内存中的对象/数组转换成可存储字符串的过程。 | `JSON.stringify(notes)` |
| **反序列化** | `Deserialization` | 将存储的字符串还原回内存中对象/数组的过程。 | `JSON.parse(...)` |
| **持久化** | `Persistence` | 确保程序在重新启动后仍能访问到之前保存的数据。 | 整个 `saveToLocal` 函数的逻辑 |

---

## 二、 运作原理：JSON 翻译官 🤖

`localStorage` 非常“死板”，它只认识 **字符串 (String)**。而你的笔记是一个 **数组 (Array)**。
为了让它们沟通，我们需要 `JSON` 这个翻译官。



### 1. 存入：打包入库 (JSON.stringify)
当你想把笔记塞进“口袋”时，必须先把它压扁成一段文本：
* **原始对象**：`[{title: "学习"}]`（活的，可操作）
* **序列化后**：`"[{'title':'学习'}]"`（死的，纯文本）
```javascript
localStorage.setItem('my_todo_data', JSON.stringify(notes));
```

### 🔑 核心方法：`localStorage.setItem()`

**1. 定义**：
用于在浏览器的本地存储空间中添加或更新一个数据项。

**2. 核心特性**：
* **语法**：('钥匙名字'，'要存的内容')
* **强制覆盖**：如果 Key（键名）已存在，新数据会直接覆盖旧数据。
* **只收字符串**：如果你直接存数组 `setItem('a', [1,2])`，它会强行把数组变成 `"1,2"`（丢失了数组特征）。所以必须配合 `JSON.stringify()` 使用。也就是说必须转化为字符串
* **容量限制**：大多数浏览器允许每个网站存 **5MB** 左右的数据（对于 TodoList 来说，存几万条笔记都够了）。

**3. 代码示例**：
```javascript
// 存入一个简单的字符串
localStorage.setItem('username', '小明');

// 存入复杂的对象（推荐做法）
const user = { id: 1, name: '小明' };
localStorage.setItem('user_info', JSON.stringify(user));
```

# 🔍 JavaScript 核心方法笔记：`filter` (过滤器)

### 1. 什么是 `filter`？
`filter` 是数组（Array）的一个内置方法，用于从数组中**筛选**出符合条件的元素。

* **形象比喻**：它就像一个“滤网”。你设定一个标准（比如：只要红色的球），把整筐球倒进去，最后它会给你一个**全新的筐**，里面只装着符合标准的球。
* **关键特性**：它**不会修改原数组**，而是返回一个过滤后的**新数组**。

---

### 2. 代码深度拆解
你提供的这行代码是一个典型的“动态过滤”逻辑，它决定了屏幕上显示哪些笔记：

```javascript
const filtered = notes.filter(n => currentView === 'all' ? !n.isDeleted : n.isDeleted);
```

### 3. 代码的语法
```javascript
const 新数组 = 原数组.filter((当前项,索引)=>{
    return n.id !== 123;
});
```
# 🎯 JavaScript 核心方法笔记：`find` (精准查找)

### 1. 什么是 `find`？
`find` 是数组（Array）的一个内置方法，用于从数组中**寻找并返回第一个**符合条件的元素。

* **形象比喻**：在一堆档案（数组）里，老师下令：“请把学号（ID）是 `1711285` 的那一张**原件**给我拿出来。”
* **关键特性**：一旦找到匹配项，它就会**立即停止**遍历，并直接返回该对象。如果翻遍了都没找到，则返回 `undefined`。

---
### 2.语法解析 
索引类似于i j的作用
```javascript
const 找到的对象 = 数组.find(索引=>数组.要找的对象 === 你要找的值)
```

---

### 3. 代码深度拆解
在你的 TodoList 项目中，这是锁定特定笔记的“雷达”：

```javascript
const users = [
    { name: 'Alice', role: 'admin' },
    { name: 'Bob', role: 'user' },
    { name: 'Charlie', role: 'user' }
];

// 查找第一个角色是 'user' 的人
const firstUser = users.find(u => u.role === 'user');

# 📍 JavaScript 核心方法：`findIndex` (索引查找)

### 1. 什么是 `findIndex`？
`findIndex` 方法用于寻找数组中满足条件的第一个元素的**索引（Index，即下标/位置）**。

* **形象比喻**：你不是在找“那个学生”，而是在找“那个学生**坐在第几排**”。
* **关键特性**：

    * **找不到时**，返回 **`-1`**（这是 JS 表示“不存在”的经典方式）。

---

```javascript
const index = array.findIndex((element, index, array) => {
  // 返回 true 则停止查找并返回当前索引
  return element.id === targetId;
});
具体语法
const index = 查找的数组.findIndex(内容(可以理解为对象 对象里面有多种属性)=>对象里面的什么属性===你想要的值)

## 一、 JavaScript 核心：数据变网页的“黄金连招”

在处理 Todo List 或任何动态列表时，这套组合拳是必修课。

### 1. `map()` —— 加工流水线
* **作用**：遍历数组，把每个元素按你的要求“变身”，返回一个**新数组**。
* **口诀**：原数组几个，新数组就几个；长度不变，内容变了。

### 2. `join()` —— 强力胶水
* **作用**：把数组里的所有项，通过指定的“连接符”粘成**一个字符串**。
* **口诀**：数组变字符串；如果不加参数 `join('')`，中间会自带逗号。注意join('')自带换行的效果

### 3. 实战案例演示
```javascript
//1.语法
// 语法结构
const 新数组 = 原数组.map((当前项, 索引, 原数组本身) => {
    return 处理后的新项;
});

// 常见简写（箭头函数）
const 新数组 = 原数组.map(item => item * 2);

// 2. 原始数据
const tasks = ['写作业', '看网课'];

// 3. 连招处理
const htmlString = tasks
    .map(item => `<li>${item}</li>`) // 变身：["<li>写作业</li>", "<li>看网课</li>"]
    .join('');                       // 粘合："<li>写作业</li><li>看网课</li>"

// 4. 注入网页
document.querySelector('#list').innerHTML = htmlString;
```

# 🚀 JavaScript `sort()` 方法深度解析笔记

`sort()` 是前端开发中处理数据的核心方法。对于计算机专业的学生来说，理解其“比较函数”的逻辑是掌握算法思维的基础。

## 1. 基础语法与默认行为
`sort()` 用于对数组元素进行原地排序（会改变原数组）。

* **语法**: `array.sort([compareFunction])`
* **默认排序**: 不传参数时，元素会被转换为 **字符串**，并按 **Unicode 码点** 排序。

> ⚠️ **警告**：直接对数字数组使用 `[1, 10, 2].sort()` 会得到 `[1, 10, 2]`，因为字符串 "10" 比 "2" 小。

---

## 2. 核心：比较函数 (Compare Function)
为了得到正确的排序结果，我们需要传入一个回调函数：`arr.sort((a, b) => { ... })`。

### 比较逻辑对照表 口诀 小不变大交换


| 返回值 (res) | 排序行为 | 记忆口诀 |
| :--- | :--- | :--- |
| **res < 0** | `a` 排在 `b` 前面 | 保持现状 (升序) |
| **res > 0** | `b` 排在 `a` 前面 | 交换位置 |
| **res === 0** | 相对位置不变 | 原地踏步 |


### 常用代码片段
```javascript
const arr = [40, 100, 1, 5, 25];

// 1. 升序 (Ascending)
arr.sort((a, b) => a - b); // [1, 5, 25, 40, 100]

// 2. 降序 (Descending)
arr.sort((a, b) => b - a); // [100, 40, 25, 5, 1]
```

### 实战例子：在处理优先级的排列时
```javascript
// 模拟数据
let notes = [
    { title: "买咖啡", priority: "low" },
    { title: "ACM 练习", priority: "high" },
    { title: "写作业", priority: "medium" }
];

// 1. 定义优先级权重
const priorityMap = {
    high: 3,
    medium: 2,
    low: 1
};

// 2. 执行排序 (高优先级排在数组前面)
notes.sort((a, b) => {
    return priorityMap[b.priority] - priorityMap[a.priority];
});

// 结果：ACM 练习(3) -> 写作业(2) -> 买咖啡(1)
